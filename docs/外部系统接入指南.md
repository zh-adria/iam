# 外部系统接入指南

本文说明外部系统如何接入 IAM 的认证与 RBAC 授权体系。

## 接入原则

- 认证使用 OAuth2/OIDC 标准协议。
- 资源服务本地校验 JWT，使用 JWKS 公钥。
- 授权优先判断权限码 `perms`，不要在业务系统里硬编码角色。
- 角色用于管理权限集合，权限码用于业务判定。
- 高敏操作可调用 token introspection 确认 token 未撤销。

## 端点

Auth Server 默认 base URL：

```text
http://localhost:8080/iam
```

常用端点：

```text
GET  /oauth/authorize
POST /oauth/token
POST /oauth/introspect
POST /oauth/revoke
GET  /oauth/jwks
GET  /oauth/.well-known/openid-configuration
GET  /oauth/userinfo
```

## Web 系统接入

Web 系统使用 Authorization Code + PKCE。

1. 在管理后台创建 OAuth2 client。
2. 配置 `redirectUris`。
3. 配置允许的 `scopes`，包含需要的权限码。
4. 浏览器跳转到 `/oauth/authorize`。
5. 回调后用 `code` 调 `/oauth/token` 换 token。
6. 资源服务使用 `access_token`。

示例 scope：

```text
openid profile order:read order:approve
```

## 后端服务接入

后端服务使用 `client_credentials`。

```bash
curl -X POST http://localhost:8080/iam/oauth/token \
  -d "grant_type=client_credentials" \
  -d "client_id=demo-client" \
  -d "client_secret=demo-secret" \
  -d "scope=order:read order:approve"
```

## JWT Claim

`access_token` 是 RS256 JWT。资源服务从 `/oauth/jwks` 拉公钥验签。

核心 claim：

```json
{
  "iss": "iam-platform",
  "sub": "admin",
  "uid": 1,
  "tenant": "default",
  "roles": ["ROLE_ADMIN"],
  "perms": ["iam:role:create"],
  "scope": "openid profile iam:role:create",
  "cid": "demo-client",
  "typ": "access",
  "jti": "..."
}
```

授权判断：

```text
允许访问 = perms 包含业务权限码
```

示例：

```text
requirePermission("order:approve")
```

## Introspection

`/oauth/introspect` 必须带 client 凭据。

```bash
curl -X POST http://localhost:8080/iam/oauth/introspect \
  -d "client_id=demo-client" \
  -d "client_secret=demo-secret" \
  -d "token=<access_token>"
```

本地 JWKS 验签无法感知 Redis 撤销状态。高敏接口建议调用 introspection。

## Authorization Check

复杂数据权限可调用 IAM 统一授权检查接口，避免外部系统复制 ABAC SpEL。

```bash
curl -X POST http://localhost:8080/iam/api/authz/check \
  -H "Authorization: Bearer <access_token>" \
  -H "Content-Type: application/json" \
  -d '{"permission":"iam:role:create","target":{"tenantCode":"default"}}'
```

响应：

```json
{
  "success": true,
  "data": {
    "allowed": true,
    "permission": "iam:role:create",
    "uid": 1,
    "sub": "admin",
    "tenant": "default"
  }
}
```

判定逻辑：

- token 必须有效。
- 当前主体必须持有 `permission`。
- 若 `auth_permission.spel_expression` 存在，继续按 `target` 做 ABAC 判断。

## RBAC 建模

推荐模型：

```text
角色 = 权限集合
权限码 = 外部系统判定依据
```

示例：

```text
ROLE_ORDER_MANAGER -> order:read, order:create, order:approve
ROLE_ORDER_VIEWER  -> order:read
```

外部系统只判断：

```text
perms contains "order:approve"
```

## SDK 边界

SDK 建议保持轻量：

- 生成 authorize URL 和 PKCE。
- 调用 token/refresh/revoke/introspect。
- JWKS 缓存与 JWT 验签。
- 解析 `IamPrincipal`。
- 提供 `hasRole` / `hasPermission` / `requirePermission`。

SDK 不应复制 IAM 内部 ABAC SpEL 逻辑。复杂数据权限应由 IAM 后续提供统一授权检查接口。

## Java SDK

当前仓库提供最小 Java SDK：

```text
backend/iam-sdk-java
```

典型用法：

```java
IamAuthClient authClient = new IamAuthClient("http://localhost:8080/iam");

IamTokenResponse token = authClient.clientCredentials(
    "demo-client",
    "demo-secret",
    "iam:role:create"
);

IamJwtVerifier verifier = new IamJwtVerifier(
    "iam-platform",
    "http://localhost:8080/iam/oauth/jwks"
);

IamPrincipal principal = verifier.verify(token.getAccessToken());
IamAuthorizer.requirePermission(principal, "iam:role:create");

IamAuthorizationCheckResponse check = authClient.checkPermission(
    token.getAccessToken(),
    "iam:role:create",
    Map.of("tenantCode", "default")
);
```

SDK 内容：

- `IamAuthClient`：OAuth2 token、refresh、introspect、revoke。
- `IamAuthClient.checkPermission`：调用 IAM 统一授权检查接口。
- `IamJwtVerifier`：JWKS 缓存和 RS256 access token 验签。
- `IamPrincipal`：统一 principal 模型。
- `IamAuthorizer`：角色和权限判断。

## Spring Boot Starter

外部 Spring Boot 资源服务可直接依赖：

```xml
<dependency>
  <groupId>com.iam</groupId>
  <artifactId>iam-sdk-spring-boot-starter</artifactId>
  <version>1.0.0-SNAPSHOT</version>
</dependency>
```

配置：

```yaml
iam:
  sdk:
    issuer: iam-platform
    auth-server-base-url: http://localhost:8080/iam
    jwks-uri: http://localhost:8080/iam/oauth/jwks
    exclude-paths:
      - /actuator/health
```

业务代码：

```java
@RestController
class OrderController {
    private final IamSecurity iam;

    OrderController(IamSecurity iam) {
        this.iam = iam;
    }

    @PostMapping("/orders/approve")
    Map<String, Object> approve() {
        iam.requirePermission("order:approve");
        return Map.of("ok", true, "user", iam.requireCurrent().getUsername());
    }
}
```

Starter 行为：

- 自动创建 `IamAuthClient`。
- 自动创建 `IamJwtVerifier`。
- 注册 Bearer token 过滤器。
- 验签成功后写入 Spring SecurityContext。
- 提供 `IamSecurity` 读取当前主体和检查权限。

## 生产要求

- 配置固定 RSA 私钥/公钥，不使用默认派生密钥。
- 配置 `iam.jwt.key-id`，并在轮换时保留旧公钥。
- OAuth2 client secret 必须安全保存。
- access token 短 TTL。
- refresh token 只放后端或安全存储。
- CORS 只放可信域名。
- introspection/revoke 必须要求 client 认证。

## RSA Key Rotation

当前 keypair：

```yaml
iam:
  jwt:
    key-id: 2026-07
    rsa-private-key-pem: ${IAM_JWT_RSA_PRIVATE_KEY_PEM}
    rsa-public-key-pem: ${IAM_JWT_RSA_PUBLIC_KEY_PEM}
```

轮换步骤：

1. 生成新 RSA PKCS#8 private key 和 X.509 public key。
2. 部署新 `key-id`、新 private/public key。
3. 把旧 public key 放入 `previous-public-keys`。
4. 等旧 access token 全部过期后移除旧 public key。

旧 key 格式：

```yaml
iam:
  jwt:
    previous-public-keys: oldKid=-----BEGIN PUBLIC KEY-----...-----END PUBLIC KEY-----
```

多个旧 key 用 `;;` 分隔：

```text
old1=PEM;;old2=PEM
```

JWKS 会同时返回当前公钥和历史公钥；签发只使用当前私钥。

## OAuth2 Client Scope UI

管理后台客户端页支持：

- 标准 OIDC scopes：`openid`、`profile`、`email`、`phone`
- 权限 scopes：从 `auth_permission` 拉取并勾选
- 自定义 scopes：保留扩展输入

外部系统接入时，client scope 应只授予该系统需要的权限码。
