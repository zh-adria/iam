## 1. 环境准备

### 必备

| 软件 | 版本 | 备注 |
|------|------|------|
| JDK | 17 | `JAVA_HOME` 指向 17 |
| Maven | 3.9+ | `mvn -v` 验证 |
| Docker | 24+ | 含 docker compose v2（可选，用于 Redis 或全栈部署） |
| Node.js | 20+ | `npm -v` 验证（构建前端用） |
| MySQL | 8.0 | 项目使用远程 MySQL：`mysql6.sqlpub.com:3311/dev_2026` |
| Redis | 7+ | 本地 `localhost:6379` 或 Docker 容器 |

Windows 环境变量（PowerShell）：
```powershell
$env:JAVA_HOME = 'D:\Program Files (x86)\jdk17'
$env:PATH = "$env:JAVA_HOME\bin;D:\Program Files (x86)\apache-maven-3.9.7\bin;$env:PATH"
```

### 加密配置密钥（必需）

项目数据库连接参数使用 `ENC()` AES-256-GCM 加密存储。启动前必须设置解密密钥：

```powershell
# 方式 1：环境变量（推荐）
$env:IAM_CONFIG_KEY = "your-secret-key-here"

# 方式 2：运行 dev.ps1 时交互输入（自动加密保存到 scripts/.secrets/）
.\scripts\dev.ps1
```

> `scripts/.secrets/` 目录被 git 忽略，不会提交到仓库。

### 生成加密配置

首次设置数据库连接时，使用加密工具生成 `ENC()` 值：

```powershell
# 设置密钥
$env:IAM_CONFIG_KEY = "your-secret-key-here"

# 运行加密脚本（默认加密远程 MySQL 连接参数）
.\scripts\encrypt-config.ps1 -ConfigKey $env:IAM_CONFIG_KEY

# 或加密自定义值
.\scripts\encrypt-config.ps1 -ConfigKey $env:IAM_CONFIG_KEY -Values "my-value-1","my-value-2"
```

将输出复制到 `application-dev.yml` / `application.yml` 中的对应位置。

### 脚本版本

每个脚本都有两个版本：
- `scripts/*.sh` — Git Bash / WSL / Linux / macOS
- `scripts/*.ps1` — Windows PowerShell（Win10/11 自带，无需 Git Bash）

| 任务 | Linux/Mac/Git Bash | Windows PowerShell |
|------|--------------------|--------------------|
| 开发模式（完整栈） | `./scripts/dev.sh` | `.\scripts\dev.ps1` |
| 仅后端 | `./scripts/dev-backend.sh` | `.\scripts\dev-backend.ps1` |
| 仅前端 | `./scripts/dev-frontend.sh` | `.\scripts\dev-frontend.ps1` |
| 加密数据库配置 | — | `.\scripts\encrypt-config.ps1` |
| 生产 Docker 部署 | `./scripts/start.sh` | `.\scripts\start.ps1` |
| 停止一切 | `./scripts/stop.sh` | `.\scripts\stop.ps1` |
| 仅构建 | `./scripts/build.sh` | `.\scripts\build.ps1` |

### Windows 首次运行 PowerShell 脚本

PowerShell 默认禁止运行 `.ps1`。两种解决方式（任选其一）：

**方式 A：永久放开当前用户（推荐，做一次）**
```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
# 之后直接运行：
.\scripts\dev.ps1
```

**方式 B：单次绕过（不改策略）**
```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\dev.ps1
```

## 2. 一键启动

### 2.1 开发模式（本地 jar + vite dev）

Linux / macOS / Git Bash：
```bash
./scripts/dev.sh
```
Windows PowerShell：
```powershell
.\scripts\dev.ps1
```

- 检查本地 Redis（`localhost:6379`），如无则尝试 Docker 启动
- 检查 `IAM_CONFIG_KEY` 环境变量（用于解密数据库配置）
- `mvn -DskipTests package` 构建后端
- 后台启动 iam-auth-server（8080，`--spring.profiles.active=dev`）
- 等 auth-server health 通过后再起 iam-admin（8081，共享同一份 MySQL 数据库）
- 前台启动 vite dev server（5173）
- Ctrl-C 一次性停掉所有进程

数据存储：远程 MySQL（`mysql6.sqlpub.com:3311/dev_2026`）。表名以 `auth_` 前缀区分功能线。
日志：`logs/auth-server.log`、`logs/admin.log`。

### 2.2 仅后端（无前端）

```bash
./scripts/dev-backend.sh     # Linux/Mac/Git Bash
.\scripts\dev-backend.ps1    # Windows PowerShell
```

### 2.3 仅前端

```bash
./scripts/dev-frontend.sh    # Linux/Mac/Git Bash
.\scripts\dev-frontend.ps1   # Windows PowerShell
```

### 2.4 生产模式（docker compose 全栈）

```bash
./scripts/start.sh           # Linux/Mac/Git Bash
.\scripts\start.ps1          # Windows PowerShell
```

前置条件：设置 `IAM_CONFIG_KEY` 环境变量。

- 构建两个后端镜像（同一 Dockerfile，`SERVICE` arg 切换）
- 启动 mysql + redis + iam-auth-server + iam-admin
- 前端单独用 nginx / CDN 部署（生产不走 vite）

### 2.5 停止一切

```bash
./scripts/stop.sh            # Linux/Mac/Git Bash
.\scripts\stop.ps1           # Windows PowerShell
```

### 2.6 仅构建不启动

```bash
./scripts/build.sh           # Linux/Mac/Git Bash
.\scripts\build.ps1          # Windows PowerShell
```

产出：
- `backend/iam-auth-server/target/boot/iam-auth-server.jar`
- `backend/iam-admin/target/boot/iam-admin.jar`
- `frontend/dist/`

## 6. 常见运维任务

### 6.1 重置管理员密码

```sql
UPDATE auth_user SET password_hash='$2a$10$...' , status=1
WHERE username='admin';
```

### 6.2 加密新的配置值

```powershell
.\scripts\encrypt-config.ps1 -ConfigKey $env:IAM_CONFIG_KEY -Values "new-value-1","new-value-2"
```

### 6.3 轮换 JWT 密钥

1. 生成新密钥（≥32 字节）：
   ```bash
   openssl rand -base64 48
   ```
2. 同时更新两个服务的 `IAM_JWT_SECRET` 环境变量。
3. 滚动重启：先重启 auth-server，再重启 admin。
4. 旧 token 在刷新时换发为新密钥签的；未刷新的 access 在原 TTL 内继续可用（如需立即失效，把旧 jti 加入 `TokenCacheService` 黑名单）。

### 6.4 解锁被锁定的用户

```sql
UPDATE auth_user SET status=1 WHERE username='alice';
```

或调 admin API：`POST /iam/admin/api/users/{id}/unlock`。

### 6.5 查看日志

```bash
# docker 部署
docker compose logs -f iam-auth-server
docker compose logs -f iam-admin

# 本地 dev
tail -f logs/auth-server.log
tail -f logs/admin.log
```

### 6.6 进入数据库

```bash
mysql -h mysql6.sqlpub.com -P 3311 -u dev_2026 -puF5RbUeRzGxb9qgG dev_2026
```

### 6.7 清空 Redis 缓存

```bash
docker exec -it iam-redis redis-cli FLUSHDB
```

> 谨慎：会清掉所有 SMS 码、Magic Token、限流计数器、access token 黑名单。

### 6.8 管理 SCIM Provisioner Token

```bash
# 创建 SCIM Bearer Token（管理态）
curl -H "Authorization: Bearer <admin-jwt>" \
  -H "Content-Type: application/json" \
  -d '{"name":"Okta Provisioner","token":"my-secret-token","tenantCode":"default"}' \
  http://localhost:8081/iam/admin/api/scim/tokens

# 列表所有 token
curl -H "Authorization: Bearer <admin-jwt>" \
  http://localhost:8081/iam/admin/api/scim/tokens

# 删除 token
curl -X DELETE -H "Authorization: Bearer <admin-jwt>" \
  http://localhost:8081/iam/admin/api/scim/tokens/{id}
```

### 6.9 管理 LDAP Group→Role 映射

管理后台入口：左侧进入「LDAP 映射」，选择租户后可新增、编辑或删除 LDAP Group DN → 角色映射。

```bash
# 创建映射：LDAP 组 DN → 角色
curl -H "Authorization: Bearer <admin-jwt>" \
  -H "Content-Type: application/json" \
  -d '{"tenantCode":"default","ldapGroupDn":"cn=developers,ou=groups,dc=example,dc=com","roleCode":"developer"}' \
  http://localhost:8081/iam/admin/api/ldap/group-mappings

# 列出租户的映射
curl -H "Authorization: Bearer <admin-jwt>" \
  "http://localhost:8081/iam/admin/api/ldap/group-mappings?tenant=default"

# 删除映射
curl -X DELETE -H "Authorization: Bearer <admin-jwt>" \
  -H "Content-Type: application/json" \
  -d '{"tenantCode":"default","ldapGroupDn":"cn=developers,ou=groups,dc=example,dc=com"}' \
  http://localhost:8081/iam/admin/api/ldap/group-mappings
```

### 6.10 注册 SAML IdP（管理态）

```bash
# 创建 IdP 注册
curl -H "Authorization: Bearer <admin-jwt>" \
  -H "Content-Type: application/json" \
  -d '{
    "tenantCode": "default",
    "registrationId": "okta",
    "idpEntityId": "http://www.okta.com/exk123456",
    "idpSsoUrl": "https://dev-123.okta.com/app/.../sso/saml",
    "idpMetadataUrl": "https://dev-123.okta.com/app/.../sso/saml/metadata",
    "spEntityId": "urn:iam:sp",
    "acsTemplate": "http://localhost:8080/iam/login/saml2/sso/{registrationId}",
    "enabled": true,
    "metadataRefreshIntervalHours": 6
  }' \
  http://localhost:8081/iam/admin/api/saml/idps

# 列表
curl -H "Authorization: Bearer <admin-jwt>" \
  "http://localhost:8081/iam/admin/api/saml/idps?tenant=default"

# 删除
curl -X DELETE -H "Authorization: Bearer <admin-jwt>" \
  http://localhost:8081/iam/admin/api/saml/idps/default/okta
```

### 6.11 配置登录页展示方式

管理后台入口：左侧进入「认证协议」→「登录页」。

| 配置项 | 说明 | 示例 |
|---|---|---|
| `iam.login.methods` | 登录页展示方式和顺序 | `password,sms,magic,social,sso,oauth2` |
| `iam.login.social-providers` | 社交登录按钮和顺序 | `wechat,qq,dingtalk,wecom` |

登录页默认突出账号密码登录；短信、Magic Link、社交登录、企业 SSO、OAuth2 会出现在「其他登录方式」按钮区，点击后按需展开。

## 9. 故障排查

### 9.1 启动报「Encrypted configuration requires IAM_CONFIG_KEY」

`application*.yml` 中包含 `ENC(...)` 加密值，但未提供解密密钥。

解决：设置 `IAM_CONFIG_KEY` 环境变量：
```powershell
$env:IAM_CONFIG_KEY = "your-secret-key"
```

### 9.2 启动报「Unable to decrypt property value: invalid key or ciphertext」

`IAM_CONFIG_KEY` 与加密时使用的密钥不一致。重新运行 `encrypt-config.ps1` 使用相同密钥。

### 9.3 启动报数据库连接失败

检查：
1. MySQL 是否可达：`mysql6.sqlpub.com:3311`
2. `IAM_CONFIG_KEY` 是否正确（影响数据库密码解密）
3. 网络连通性：`ping mysql6.sqlpub.com`

### 9.4 Liquibase 锁

异常中断后 `databasechangeloglock` 表残留锁：
```sql
UPDATE databasechangeloglock SET locked=0;
```

### 9.5 前端登录 401 但后端日志正常

检查 `vite.config.ts` 代理：`/iam/admin/*` → 8081，`/iam/*` → 8080。顺序不能反（更具体的在前）。

### 9.6 admin 接口 403

JWT 里没有 `ROLE_ADMIN` 角色。用 admin 账号登录（`admin/Iam@2026`），不要用 alice。

### 9.7 SAML 启动报「no RelyingPartyRegistration」

`iam.saml.idp.metadata-url` 为空，已提供占位 registration，仅 WARN 不阻塞启动。真实接入填 metadata URL 即可。

### 9.8 LDAP 启动报「LDAP not configured」

`iam.ldap.url` 为空时禁用 LDAP 登录。租户级 LDAP 在 `tenants.ldap_url` 配置。

### 9.9 OAuth2 测试

```bash
# 授权码流程（浏览器打开）
http://localhost:8080/iam/oauth/authorize?response_type=code&client_id=demo-client&redirect_uri=http://localhost:5173/callback&scope=openid

# 换 token
curl -X POST http://localhost:8080/iam/oauth/token \
  -d 'grant_type=authorization_code&code=<CODE>&redirect_uri=http://localhost:5173/callback&client_id=demo-client&client_secret=demo-secret'

# introspect
curl -X POST http://localhost:8080/iam/oauth/introspect \
  -d 'client_id=demo-client&client_secret=demo-secret&token=<ACCESS_TOKEN>'
```

### 9.10 SCIM 401

```bash
# 检查 Bearer Token 是否正确
curl -v -H "Authorization: Bearer <token>" http://localhost:8080/iam/scim/v2/Users
# 管理态查看 token：GET /iam/admin/api/scim/tokens
```

## 10. 升级与回滚

- 升级前：`./scripts/build.sh` 全量构建验证 + `cd backend && mvn test`。
- 数据库变更走 Liquibase changeset，不手改 schema。
- 回滚：Liquibase `rollback` + git checkout 旧代码 + 重启。
- 蓝绿：两套 docker-compose stack，切 nginx upstream。

## 11. Windows 开发实战

### 11.1 一键脚本命令速查

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned   # 仅首次
.\scripts\dev.ps1            # 启动 dev（redis + backend + vite）
.\scripts\dev-backend.ps1    # 仅启动后端
.\scripts\dev-frontend.ps1   # 仅启动前端
.\scripts\start.ps1          # Docker 部署
.\scripts\stop.ps1           # 停止全部
.\scripts\build.ps1          # 仅构建，不启动
.\scripts\encrypt-config.ps1 # 加密配置值
```

### 11.2 加密配置

**首次设置：**
```powershell
# 1. 设置你的密钥
$env:IAM_CONFIG_KEY = "my-super-secret-key-2026"

# 2. 运行加密脚本（自动编译工具类并加密远程 MySQL 连接参数）
.\scripts\encrypt-config.ps1 -ConfigKey $env:IAM_CONFIG_KEY

# 3. 将输出的 ENC(...) 值复制到 application-dev.yml 和 application.yml
```

**持久化密钥（避免每次重启输入）：**
```powershell
# 保存到 scripts/.secrets/iam-config-key.txt（Windows DPAPI 加密）
# dev.ps1 首次运行时交互输入后自动保存
```

**手动加密单个值：**
```powershell
.\scripts\encrypt-config.ps1 -ConfigKey $env:IAM_CONFIG_KEY -Values "my-value"
```

**Q：`vite dev` 起不来？**
A：确认已 `cd frontend && npm i`。确认 Node.js ≥ 20（`node -v`）。

**Q：后端报「IAM_CONFIG_KEY is required」？**
A：设置环境变量：`$env:IAM_CONFIG_KEY = "your-key"`，或运行 `.\scripts\encrypt-config.ps1` 配置加密值。

**Q：后端报「Unable to decrypt property value」？**
A：`IAM_CONFIG_KEY` 与加密时使用的密钥不一致。重新使用相同密钥加密。

**Q：后端报数据库连接失败？**
A：检查 MySQL 是否可达（`mysql6.sqlpub.com:3311`），以及 `IAM_CONFIG_KEY` 是否正确（影响密码解密）。

**Q：前端白屏，浏览器控制台报 `hasRole is not a function`？**
A：`npm run dev` 需要和 backend 同起（`.\scripts\dev.ps1`），否则 `/iam/admin/*` 反向代理找不到 admin 服务。

**Q：改了 `global.css`，浏览器没生效？**
A：Ctrl+Shift+R 强刷。Vite HMR 会导入 CSS，但若切换过 `.ps1` 脚本重启，需要清浏览器缓存。

**Q：SCIM 端点 401？**
A：`iam.scim.auth-token` 未配置或 Bearer Token 不正确。管理态 `POST /iam/admin/api/scim/tokens` 创建 token。

**Q：SAML 启动报「no RelyingPartyRegistration」？**
A：`auth_saml_idp_registration` 表无启用记录，且 `application.yml` 未配置 `iam.saml.idp.metadata-url`。管理态创建 IdP 注册即可。

## 12. 安全 checklist（上线前）

- [ ] `IAM_CONFIG_KEY` 已设为强随机密钥（≥16 字符），不提交到代码仓库
- [ ] `IAM_JWT_SECRET` 已改成 ≥32 字节随机值，两服务相同
- [ ] `application.yml` 中所有 `ENC()` 值已用生产密钥重新加密
- [ ] `DB_PASS` 已改非默认
- [ ] `REDIS_PASS` 已设置
- [ ] `iam.cors.allowed-origins` 已改为生产域名
- [ ] 演示账号 `admin/Iam@2026` 密码已改或已禁用
- [ ] `DemoSeeder` 在生产 profile 下不执行（移除 `@Component` 或加 `@Profile("dev")`）
- [ ] actuator 除 health 外不对外暴露
- [ ] HTTPS 终端（nginx / 反向代理 TLS）
- [ ] 真实 SMS / 邮件通道已接入（替换 stub）
- [ ] 审计日志定期归档
- [ ] SCIM Provisioner Token 已创建（`iam.scim.auth-token`），非默认值
- [ ] LDAP Group→Role 映射按最小权限原则配置
